Interface

In object-oriented programming, an interface is a blueprint for a class or a group of related classes. It defines a set of methods or behaviors that the class or classes implementing the interface must provide. The interface does not provide any implementation of the methods, but merely specifies the signature or the prototype of the methods that the implementing classes must define.

The purpose of an interface is to provide a common protocol or a contract for communication between different classes. By defining a common interface, multiple classes can interact with each other without knowing the internal details of each other's implementation. This helps to promote code reuse, maintainability, and flexibility.

In Java, an interface is defined using the interface keyword, and the methods declared in the interface are implicitly abstract and public. A class can implement one or more interfaces using the implements keyword, and it must provide an implementation for all the methods declared in the interface. A class can also extend another class and implement one or more interfaces at the same time.

In summary, an interface is a powerful feature of object-oriented programming that allows for abstraction, decoupling, and polymorphism. It defines a contract between classes and provides a mechanism for loose coupling and high cohesion.

Inheritance

Inheritance is a fundamental concept of object-oriented programming (OOP) that allows new classes to be based on existing classes, thereby inheriting the properties and behavior of the parent class. The parent class is also known as the superclass or base class, and the child class is known as the subclass or derived class.

Inheritance allows the reuse of code, reducing duplication and improving code maintainability. The subclass can inherit attributes, methods, and properties from the superclass, and it can also add its own attributes and methods or override the existing ones. This allows for code customization and flexibility.

In Java, inheritance is achieved using the extends keyword. A subclass can extend only one superclass, but a superclass can have multiple subclasses. The subclass inherits all the non-private members of the superclass, including its constructors, methods, and instance variables.

Inheritance also facilitates polymorphism, which allows objects of different classes to be treated as if they belong to the same class. This enables code flexibility and reusability and can make code more modular and easier to maintain.

However, inheritance can also lead to code complexity and tight coupling between classes if not used judiciously. Therefore, it is essential to design class hierarchies carefully and consider other OOP principles such as composition, aggregation, and interface implementation when creating classes.

Encapsulation

Encapsulation is a fundamental concept of object-oriented programming (OOP) that refers to the bundling of data and methods within a single unit, such as a class. The primary purpose of encapsulation is to ensure that the data and methods are not accessible outside the class except through the defined methods, also known as accessors or mutators.

In Java, encapsulation is achieved using the access modifiers - private, public, protected, and default (also known as package-private). The private modifier restricts access to the class itself, while the public modifier allows unrestricted access from outside the class. The protected modifier allows access from within the same package or from a subclass, and the default modifier allows access within the same package.

Encapsulation has several benefits. It provides data security by preventing unauthorized access to the data, which can help prevent data corruption or tampering. It also enhances code maintainability and reusability by making it easier to change the internal implementation of the class without affecting other parts of the code.

Encapsulation also promotes code readability and reduces code complexity by hiding the implementation details from the outside world. It enables the creation of more modular and flexible code, which can be easier to understand and maintain.

Overall, encapsulation is an essential principle of OOP that can help create more secure, maintainable, and flexible code.

Polymorphism

Polymorphism is a fundamental concept of object-oriented programming (OOP) that allows objects of different classes to be treated as if they belong to the same class. Polymorphism enables code flexibility and reusability and can make code more modular and easier to maintain.

In Java, polymorphism can be achieved through method overloading and method overriding. Method overloading involves defining multiple methods with the same name but different parameters in a class. The compiler determines which method to call based on the arguments passed at runtime.

Method overriding involves redefining a method in the subclass with the same name, return type, and parameters as in the superclass. When the method is called using a reference variable of the superclass, the subclass implementation is executed.

Polymorphism also involves the use of inheritance and interfaces. A superclass reference variable can be used to refer to objects of a subclass, allowing for code reuse and flexibility. An interface reference variable can be used to refer to objects of different classes that implement the same interface, enabling loose coupling and high cohesion.

Polymorphism can also be achieved through generic programming, which involves creating classes or methods that can work with different data types, allowing for greater code reuse and flexibility.

Overall, polymorphism is a powerful concept that allows for greater code flexibility, reusability, and maintainability in object-oriented programming. It enables objects of different classes to be treated as if they belong to the same class, providing a mechanism for code modularity and extensibility.

Bank as Interface

The Bank interface is a set of methods that define the basic functionalities that a bank should provide. The interface specifies methods for checking account balance, adding and withdrawing money, changing password, and calculating the total interest. These methods take input parameters such as the password, amount of money, old and new password, and the number of years for calculating the interest.

In addition to the specified methods, the interface also mentions a comment regarding transferring money within the same bank. While this functionality is not explicitly defined in the interface, it suggests that it is an important feature that banks should provide.

Implementing this interface allows banks, such as SBI and HDFC, to define their own classes and implement these methods according to their specific requirements. This interface serves as a blueprint for defining the basic functionality of a bank and enables code reuse and flexibility.

SBI implements Bank

The SBI class is an implementation of the Bank interface and provides the functionality specific to the State Bank of India (SBI). This class defines methods for checking balance, adding and withdrawing money, changing the password, and calculating the total interest for a given number of years. Here are some points to understand the SBI class:

  • The SBI class has private instance variables, including balance, account number, password, and name.
  • The constructor of the SBI class takes in the initial balance, password, and account holder name as parameters and generates a random UUID to assign to the account number.
  • The checkBalance method overrides the method from the Bank interface and checks if the password provided matches the account password before returning the account balance.
  • The addMoney method overrides the Bank interface method and adds the provided amount to the account balance, returning a message that confirms the transaction.
  • The withdrawMoney method overrides the Bank interface method and withdraws the provided amount from the account balance if the password matches and the account has sufficient funds.
  • The changePassword method overrides the Bank interface method and changes the password if the old password matches.
  • The calculateTotalInterest method overrides the Bank interface method and calculates the total interest for the account balance for the given number of years at the SBI rate of interest.

Overall, the SBI class provides the necessary functionality to interact with an SBI bank account and demonstrates the use of interface methods to define common functionality across different bank implementations.

HDFC implements Bank

HDFC Bank is another implementation of the Bank interface, similar to SBI class. HDFC class also has the following attributes and methods:

Attributes:

  • balance: an integer to store the current balance of the account.
  • accNumber: a string to store the account number generated by a random unique identifier
  • password: a string to store the password of the account.
  • name: a string to store the name of the account holder.
  • rateOfInterest: a static double variable to store the rate of interest offered by the bank.

Methods:

  • checkBalance(String password): a method to check the balance of the account by passing the password. It returns the balance if the password is correct, otherwise, it returns -1.
  • addMoney(int money): a method to add money to the account. It takes an integer argument as money and updates the balance accordingly. It returns a string message indicating the transaction details.
  • withdrawMoney(int drawMoney, String password): a method to withdraw money from the account. It takes an integer argument as the amount to withdraw and the account password. It checks if the password is correct and the balance is sufficient to withdraw the amount. If the conditions are satisfied, it updates the balance and returns a success message; otherwise, it returns an error message.
  • changePassword(String oldPassword, String newPassword): a method to change the password of the account. It takes two string arguments as old and new passwords. If the old password matches the current password, it updates the password and returns a success message; otherwise, it returns an error message.
  • calculateTotalInterest(int years): a method to calculate the total interest based on the balance and the rate of interest over the specified number of years. It takes an integer argument as the number of years and returns the calculated interest.

Overall, the HDFC class is another implementation of the Bank interface that provides functionality for checking balance, adding and withdrawing money, changing password, and calculating interest.

Main function

The application allows users to choose between two banks, SBI and HDFC, and perform basic banking operations such as checking account balance, adding and withdrawing money, calculating interest, and changing passwords.

The main class contains the main method and the choose_bank method, which displays the options for the user to choose between the two banks. Depending on the user's choice, the create_accountSBI or create_accountHDFC method is called to create a new account for the respective bank.

Both create_accountSBI and create_accountHDFC methods prompt the user to enter their name, initial balance, and password, and create a new account object for the respective bank with the user's input.

After creating the account, the user is presented with a menu of options to choose from, such as checking the account balance, adding or withdrawing money, calculating interest, changing the account password, or exiting the application.

Based on the user's choice, the corresponding method is called to perform the desired operation. For example, if the user chooses to check their balance, the checkBalance method is called and the account balance is displayed. Similarly, if the user chooses to add or withdraw money, the addMoney or withdrawMoney method is called with the user's input.

Overall, this application provides a simple example of how to implement basic banking operations in Java using object-oriented programming concepts.
